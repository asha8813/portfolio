04 / How I think

Four questions every AI/ML visual has to answer.

The principles I carry into every model diagram, dashboard, and dataset annotation.

T / 01

Hierarchy

Input → model → output. I design layers of explanation: the headline insight, supporting evidence, and optional detail. If you only read the top layer, you still understand the architecture.

T / 02

Readability

Metrics, tensors, and thresholds need tabular figures, consistent decimals, and no decorative clutter. Two typefaces max. One accent for actionable signals.

T / 03

Storytelling

Every model evaluation is an argument. Before designing, I write the "so what" in one sentence: what should the viewer do after seeing this?

T / 04

Simplification

What's the minimum necessary to show the truth? Remove every embellishment that doesn't serve understanding. Honesty over decoration.

Rigor beats novelty. Clarity beats complexity.
